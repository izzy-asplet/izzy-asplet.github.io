THE DOM

The DOM (Document Object Model) is an object oriented representation of a web page made up of nodes such as elements, attributes and text content and represents the document as a tree structure. It allows the developer methods for adding and removing nodes, moving them to different locations in the tree, or modifying their contents.

ARRAY VS. OBJECTS

An array is a data collection of a single variable of a similar type. Arrays start counting from 0. In the example below “tomato” would be [1] and “sardines” would be [3]. Items can be modified with using push(), pop(), shift(), unshift() methods.

Objects on the other hand are made up of a “thing” that can have many characteristics. These characteristic are called properties that consist of a key and a value. Properties can be modified by using dot or bracket notations.

FUNCTIONS AND THEIR USE

A function is a set of code designed to perform a certain task when it is called. It must be defined first in the scope before calling it. You’re able to pass a value to the function and it will return a value. By having a function you are able to write a set of instructions once rather than writing it every time you want the program to do something.
